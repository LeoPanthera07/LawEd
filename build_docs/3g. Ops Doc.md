## 1. Deployment Architecture Overview

NyayaSetu MVP runs across two managed platforms — Vercel for the frontend and Railway for all backend services. This eliminates DevOps overhead for a solo builder while maintaining production-grade reliability. Every service is containerized and independently deployable.

```

    │
```
```
    ▼
```

### 4.2 GitHub Actions Workflow

```yaml
# .github/workflows/deploy.yml
name: NyayaSetu CI/CD

on:
  push:
    branches: [main, staging]
  pull_request:
    branches: [main]

env:
  PYTHON_VERSION: "3.11"
  NODE_VERSION: "20"

jobs:
  test-backend:
    name: Backend Tests
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_DB: nyayasetu_test
          POSTGRES_USER: test_user
          POSTGRES_PASSWORD: test_pass
        ports:
          - 5433:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

      qdrant:
        image: qdrant/qdrant:v1.8.0
        ports:
          - 6334:6333

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: pip

      - name: Install dependencies
        run: |
          cd backend
          pip install -r requirements.txt
          pip install -r requirements-dev.txt

      - name: Lint with ruff
        run: |
          cd backend
          ruff check .

      - name: Type check with mypy
        run: |
          cd backend
          mypy . --ignore-missing-imports

      - name: Run unit tests
        env:
          TEST_DATABASE_URL: postgresql+asyncpg://test_user:test_pass@localhost:5433/nyayasetu_test
          QDRANT_HOST: localhost
          QDRANT_PORT: 6334
          LLM_PROVIDER: mock
          APP_SECRET_KEY: test-secret-key-32-bytes-minimum
        run: |
          cd backend
          pytest tests/backend/unit/ -v \
            --cov=. \
            --cov-report=xml \
            --cov-fail-under=85 \
            -m "not integration and not performance and not e2e"

      - name: Upload coverage
        uses: codecov/codecov-action@v4
        with:
          file: ./backend/coverage.xml

  test-frontend:
    name: Frontend Tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: npm
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: |
          cd frontend
          npm ci

      - name: Type check
        run: |
          cd frontend
          npm run type-check

      - name: Lint
        run: |
          cd frontend
          npm run lint

      - name: Build
        run: |
          cd frontend
          npm run build

  deploy-staging:
    name: Deploy to Staging
    needs: [test-backend, test-frontend]
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/staging'

    steps:
      - uses: actions/checkout@v4

      - name: Deploy to Railway Staging
        uses: bervProject/railway-deploy@main
        with:
          railway_token: ${{ secrets.RAILWAY_TOKEN }}
          service: nyayasetu-staging

  deploy-production:
    name: Deploy to Production
    needs: [test-backend, test-frontend]
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v4

      - name: Deploy to Railway Production
        uses: bervProject/railway-deploy@main
        with:
          railway_token: ${{ secrets.RAILWAY_TOKEN }}
          service: nyayasetu-production

      - name: Notify deployment
        uses: slackapi/slack-github-action@v1
        with:
          payload: |
            {
              "text": "NyayaSetu production deployment complete ✅\nCommit: ${{ github.sha }}\nBy: ${{ github.actor }}"
            }
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

### 6.3 Health Check Endpoints

```python
# backend/api/v1/health.py

@router.get("/health")
async def basic_health():
    """Public health check — used by Railway and load balancers"""
    return {"status": "healthy", "timestamp": datetime.utcnow().isoformat()}

@router.get("/health/deep")
async def deep_health(admin=Depends(require_admin)):
    """
    Deep health check — verifies all dependencies.
    Admin only.
    """
    checks = {}

    # PostgreSQL
    try:
        await db.execute("SELECT 1")
        checks["postgresql"] = "healthy"
    except Exception as e:
        checks["postgresql"] = f"unhealthy: {str(e)}"

    # Qdrant
    try:
        await qdrant_client.get_collection("nyayasetu_clauses")
        checks["qdrant"] = "healthy"
    except Exception as e:
        checks["qdrant"] = f"unhealthy: {str(e)}"

    # Neo4j
    try:
        await neo4j_client.run("RETURN 1")
        checks["neo4j"] = "healthy"
    except Exception as e:
        checks["neo4j"] = f"unhealthy: {str(e)}"

    # LLM
    try:
        await llm_client.complete("ping", max_tokens=5)
        checks["llm"] = "healthy"
    except Exception as e:
        checks["llm"] = f"unhealthy: {str(e)}"

    overall = "healthy" if all(
        v == "healthy" for v in checks.values()
    ) else "degraded"

    return {
        "status": overall,
        "checks": checks,
        "timestamp": datetime.utcnow().isoformat()
    }
```

- Deploy to staging
- Run full QA sign-off suite against staging

- Invite-only: 20–50 trusted users from IIT Gandhinagar network

**Phase 2 — Open Beta (Week 2–4)**

## 9. Feature Flags

Feature flags allow enabling or disabling features without redeployment. Implemented as environment variables for MVP — no external service needed.

```python
# backend/config.py

class FeatureFlags:
    # Controls whether writeup generation is available
    WRITEUP_ENABLED: bool = os.getenv("FF_WRITEUP_ENABLED", "true") == "true"

    # Controls whether guest mode (no-login query) is active
    GUEST_MODE_ENABLED: bool = os.getenv("FF_GUEST_MODE", "true") == "true"

    # Controls whether Neo4j graph traversal runs
    # Can be disabled if Neo4j has issues — degrades to Qdrant-only
    GRAPH_TRAVERSAL_ENABLED: bool = os.getenv("FF_GRAPH_TRAVERSAL", "true") == "true"

    # Controls payment requirement for writeup
    # Set to false for internal testing only — never in production
    PAYMENT_REQUIRED: bool = os.getenv("FF_PAYMENT_REQUIRED", "true") == "true"

    # Controls human escalation routing
    ESCALATION_ENABLED: bool = os.getenv("FF_ESCALATION_ENABLED", "true") == "true"

flags = FeatureFlags()
```

### 11.3 Corpus Rollback (Qdrant)

```bash
# Restore Qdrant collection from snapshot
curl -X PUT \
  "http://${QDRANT_HOST}:${QDRANT_PORT}/collections/nyayasetu_clauses/snapshots/recover" \
  -H "Content-Type: application/json" \
  -d '{"location": "<snapshot_path>"}'
```

### 11.4 Rollback Decision Tree

```
Issue detected
      │
      ▼
Is it a frontend issue only?
  YES → Rollback Vercel deployment (30 seconds)
  NO  ↓
      ▼
Is it a backend bug (not data)?
  YES → Rollback Railway FastAPI deployment (60 seconds)
  NO  ↓
      ▼
Is it a database migration issue?
  YES → Run alembic downgrade -1, then rollback backend
  NO  ↓
      ▼
Is it a corpus / Qdrant issue?
  YES → Restore Qdrant snapshot, restart FastAPI
  NO  ↓
      ▼
Is it an external service issue (Groq, Razorpay)?
  YES → Enable fallback provider via feature flag, monitor
  NO  ↓
      ▼
Escalate to manual investigation
Log incident report
```

## 12. Incident Response

### 12.1 Severity Classification

|Severity|Definition|Response Time|Example|
|---|---|---|---|
|P0 — Critical|Platform completely down, payments failing, data loss|15 minutes|PostgreSQL unreachable, all queries failing|
|P1 — High|Core feature broken, >10% query failure rate, hallucination detected|1 hour|LLM API down, escalation routing broken|
|P2 — Medium|Degraded performance, elevated latency, single feature broken|4 hours|Neo4j traversal failing (graceful degradation active)|
|P3 — Low|Minor UX issue, non-critical feature broken|Next business day|Analytics dashboard not loading|

### 12.2 Incident Response Process

```
1. DETECT
   Alert received (email/UptimeRobot) or noticed manually
   │
   ▼
2. ASSESS
   Check Railway logs → identify affected component
   Check /health/deep endpoint
   Classify severity (P0/P1/P2/P3)
   │
   ▼
3. COMMUNICATE
   P0/P1: Post status update on nyayasetu.in/status
   Notify evaluator lawyers if escalation queue affected
   │
   ▼
4. MITIGATE
   Apply quickest available fix:
   - Rollback deployment
   - Disable feature flag
   - Switch LLM provider
   - Restart service
   │
   ▼
5. RESOLVE
   Verify fix via /health/deep
   Run smoke test: 3 golden queries
   Monitor for 15 minutes post-fix
   │
   ▼
```

## 13. Corpus Update Process

When BNS, BNSS, or BSA is amended by Parliament, the corpus must be updated without full system downtime.

```bash
#!/bin/bash
# scripts/update_corpus.sh
# Run this when a law amendment is published

echo "Starting corpus update process..."

# Step 1: Snapshot current Qdrant state
echo "Creating Qdrant snapshot..."
curl -X POST "http://${QDRANT_HOST}:6333/collections/nyayasetu_clauses/snapshots"

# Step 2: Create Neo4j backup
echo "Backing up Neo4j..."
neo4j-admin database dump neo4j --to-path=/tmp/neo4j_pre_update.dump

# Step 3: Run PDF ingestion for updated document
echo "Ingesting updated PDF..."
python scripts/run_pdf_ingestion.py --act BNS --pdf ./data/BNS_amended.pdf

# Step 4: Re-embed changed sections only
echo "Re-embedding changed sections..."
python scripts/run_embedding_pipeline.py --incremental --act BNS

# Step 5: Re-extract graph relationships for affected sections
echo "Updating graph relationships..."
python scripts/run_graph_extraction.py --incremental --act BNS

# Step 6: Run 20 golden queries to validate
echo "Running retrieval quality validation..."
pytest tests/data_pipeline/test_retrieval_quality.py -v

# Step 7: If validation passes, corpus update complete
echo "Corpus update complete. Monitor query quality for 24 hours."
```
