## 1. Product Overview

NyayaSetu is a web-first legal intelligence platform for Indian citizens. It accepts a natural language description of any legal situation — in English, Hindi, or Hinglish — and returns a structured legal intelligence response mapping the situation to relevant clauses and articles across BNS (Bharatiya Nyaya Sanhita), BNSS (Bharatiya Nagarik Suraksha Sanhita), and BSA (Bharatiya Sakshya Adhiniyam).

NyayaSetu is not a chatbot. It is not a search engine. It is a single-input, single-output legal intelligence tool — the digital entry point to the Indian justice system.

---

## 2. Goals and Objectives

### Product Goals

- Enable any Indian citizen to understand which laws apply to their situation in under 5 minutes
- Reduce the cost of initial legal orientation from ₹2,000–₹10,000 to ₹10–₹29
- Generate structured, actionable legal documents (FIR-ready / lawyer-ready writeups) that citizens can act on immediately
- Provide a human escalation path for high-stakes or low-confidence cases
- Establish a foundation that scales into a full legal action pipeline in later phases

### Non-Goals for MVP

- Voice input (deferred to Phase 2)
- State-specific law amendments (deferred to Phase 2)
- Regional languages beyond English, Hindi, Hinglish (deferred to Phase 2)
- Agentic plaintiff/defendant debate system (deferred to Phase 3)
- Indian judiciary public records integration (deferred to Phase 3)
- Automated lawyer matching (deferred to Phase 2)
- Mobile native apps (deferred to Phase 2)

## 4. User Stories

### 4.1 Citizen User Stories

**US-01 — Submit a legal query** As a citizen, I want to describe my legal situation in plain language so that I can understand which laws apply to my case without needing a lawyer.

Acceptance criteria:

- Input field accepts free-form text in English, Hindi, or Hinglish
- Minimum input length: 20 characters
- Maximum input length: 5,000 characters
- Input is submitted via a single action (button click or keyboard shortcut)
- System acknowledges receipt and shows a processing indicator within 500ms

---

**US-02 — Receive structured legal intelligence output** As a citizen, I want to see a clear, structured response showing which laws apply to my situation so that I can understand my rights and options.

Acceptance criteria:

- Response displays within 15 seconds of submission for 95% of queries
- Response shows 5–10 most relevant clauses ranked by relevance
- Each clause entry shows: Act name, Section number, Section title, exact statutory text, plain-English explanation, relevance score
- Response is displayed in the same language as the input (Hindi query → Hindi explanation)
- Each clause is expandable — collapsed by default showing section number and title, expanded showing full text and explanation
- Response includes a disclaimer: "This is legal information, not legal advice. Consult a qualified lawyer for your specific situation."

---

**US-03 — Purchase a structured writeup** As a citizen, I want to generate a formal written document of my situation and applicable laws so that I can submit it directly to a police station or law firm.

Acceptance criteria:

- Writeup generation is offered as a paid action after clause mapping is displayed
- Price is shown clearly before payment is initiated (₹99–₹299 depending on complexity)
- Payment is processed via Razorpay — UPI, card, and net banking supported
- Writeup is generated within 30 seconds of payment confirmation
- Writeup is downloadable as a PDF
- Writeup contains: incident summary (structured), applicable acts and clauses with exact text, citizen details (name, contact — entered at writeup generation step), date and time of incident, recommended next steps (approach police / approach lawyer / file complaint)
- Writeup is stored in citizen's case history and re-downloadable at any time

---

**US-04 — View case history** As a citizen, I want to view all my previous queries and writeups so that I can refer back to them or share them.

Acceptance criteria:

- Case history is accessible from the citizen dashboard
- Each entry shows: query date, query summary (first 100 characters), status (mapped / escalated / writeup generated), writeup download link if applicable
- History is paginated — 10 entries per page
- Citizen can delete their own case history entries

---

**US-05 — Account registration and login** As a citizen, I want to create an account so that my queries and writeups are saved and I can access them later.

Acceptance criteria:

- Registration accepts: name, email, password, phone number (optional)
- Email verification required before first query
- Login via email + password
- Password reset via email OTP
- Session persists for 7 days unless manually logged out
- Guest mode allowed for one free query without registration — registration required for history and writeup

---

**US-06 — Free tier query** As a first-time citizen, I want to try NyayaSetu without paying so that I can evaluate whether it is useful before committing.

Acceptance criteria:

- 3 free clause mapping queries per month per registered account
- 1 free query allowed in guest mode (no account)
- Free tier does not include writeup generation
- Free tier usage counter is displayed to the user (e.g. "2 of 3 free queries used this month")
- After free tier is exhausted, user is shown a clear upgrade prompt with pricing

---

### 4.2 Evaluator Lawyer User Stories

**US-07 — View escalated case queue** As an evaluator lawyer, I want to see all cases escalated for human review so that I can validate the AI output and ensure accuracy.

Acceptance criteria:

- Dashboard shows all escalated cases in a queue, sorted by escalation time (oldest first)
- Each case entry shows: escalation reason (low confidence / high stakes), query summary, AI-generated clause map, confidence score, citizen anonymized ID
- Cases are filterable by escalation reason and date
- Case queue updates in real time without page refresh

---

**US-08 — Review and validate a case** As an evaluator lawyer, I want to review the AI-generated clause map for an escalated case and either approve it, correct it, or flag it so that citizens receive accurate legal information.

Acceptance criteria:

- Lawyer can view full citizen query and full AI-generated response
- Lawyer can approve the response as-is
- Lawyer can edit the clause list — add clauses, remove clauses, reorder by relevance
- Lawyer can add a legal note (free text, max 500 characters) visible to the citizen
- Lawyer can flag a case as requiring further review (escalates to Admin)
- All lawyer actions are timestamped and logged
- Citizen is notified when their escalated case has been reviewed

---

**US-09 — Evaluator lawyer login and dashboard** As an evaluator lawyer, I want a dedicated login and dashboard so that I can access only the cases assigned to my review queue.

Acceptance criteria:

- Evaluator lawyer login is separate from citizen login — role is assigned by Admin
- Dashboard shows: pending cases count, reviewed cases count, average review time, cases reviewed today
- Lawyer cannot access citizen personal details beyond anonymized ID
- Session expires after 8 hours of inactivity

---

### 4.3 Admin User Stories

**US-10 — User management** As an admin, I want to manage all platform users so that I can assign roles, deactivate accounts, and monitor platform health.

Acceptance criteria:

- Admin dashboard shows total users by role, new registrations per day, active sessions
- Admin can assign or revoke Evaluator Lawyer role
- Admin can deactivate any citizen or lawyer account
- Admin can view full case history for any user (audit purpose)

---

**US-11 — Platform analytics** As an admin, I want to see usage analytics so that I can monitor product health and identify issues.

Acceptance criteria:

- Analytics dashboard shows: total queries per day, average response time, escalation rate, writeup conversion rate, revenue per day
- Data is refreshed every 15 minutes
- Admin can export analytics as CSV for any date range

## 8. User Flow — Primary Path

```
1. Citizen lands on NyayaSetu homepage
2. Citizen enters legal situation in natural language (text area)
3. Citizen clicks "Analyse My Situation"
4. System shows processing indicator (estimated time: 10–15 seconds)
5. System returns structured clause map:
   - Top 5–10 clauses ranked by relevance
   - Each clause: Act | Section | Title | Statutory Text | Plain Explanation | Relevance
   - Disclaimer shown at bottom
6. Citizen reviews clause map
7. System shows "Generate Formal Document" CTA (paid action)
8. Citizen clicks → enters name and contact details → Razorpay payment flow
9. Payment confirmed → writeup generated → PDF downloaded
10. Writeup stored in citizen case history
```

---

## 9. User Flow — Escalation Path

```
1. Citizen submits query
2. LangGraph confidence evaluator scores mapping below threshold
   OR detects high-stakes indicators (life, liberty, serious harm)
3. System routes case to human escalation node
4. Citizen receives message:
   "Your situation involves complex legal considerations.
    A verified legal expert is reviewing your case.
    You will be notified within [X hours]."
5. Case appears in evaluator lawyer dashboard
6. Evaluator lawyer reviews AI output, approves / edits / flags
7. Citizen notified via email: "Your case has been reviewed"
8. Citizen views updated clause map with lawyer's notes
9. Writeup generation now available (if lawyer approved)
```

## 11. Constraints

### Technical Constraints

- Open source and free infrastructure only for MVP — no paid cloud AI APIs until revenue justifies it
- LLM: Groq + Llama 3 for MVP — Claude API switchover is a configuration change, not a rebuild
- English, Hindi, and Hinglish only — multilingual-e5-large embedding model handles all three
- Text input only — voice deferred to Phase 2
- Central laws (BNS, BNSS, BSA) only — state amendments deferred

### Business Constraints

- Solo builder — architecture must be modular, each component independently buildable and testable
- Zero external capital for MVP — all infrastructure must be free or open source tier
- Legal disclaimer required on every output — product provides information, not advice
- DPDP Act 2023 compliance required — citizen data handling must meet India's data protection standards

### Scope Constraints

- MVP does not include the agentic plaintiff/defendant debate system
- MVP does not include judiciary records integration
- MVP does not include automated lawyer matching
- MVP does not include the unique case ID system for law firm intake (Phase 2)
